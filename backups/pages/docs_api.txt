Want to add a [url=http://www.django-rest-framework.org/]django-rest-framework[/url] API that developers can use to integrate your forum into their software with?  So did I.  Django Boards [url=https://github.com/shakedown-street/django-boards/tree/dev/django_boards/rest]comes shipped[/url] with a django-rest-framework implementation for accessing exactly what data you'd be able to access if you were browsing the forum itself.

You need only to configure project level settings to be able to implement a full rest API.

[ol]
[li]Add these apps into [b]INSTALLED_APPS[/b] in [b]settings.py[/b]:
[code]INSTALLED_APPS = [
    ...
    'rest_framework',
    'rest_framework.authtoken',
    'corsheaders',
    ...
    'django_boards', # To be safe, I'd make sure to add this below the rest_framework apps
][/code]
[/li]
[li]Django Boards provides the serializers, permissions, and viewsets for you, but you still have control over the project level settings.

Add this anywhere, also in [b]settings.py[/b]:
[code]CORS_ORIGIN_ALLOW_ALL = True

REST_FRAMEWORK = {                        
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework.authentication.TokenAuthentication',
        'django_boards.rest.authentication.NoCSRFSessionAuthentication',
    ),
    'DEFAULT_PAGINATION_CLASS': 'rest_framework.pagination.PageNumberPagination',
    'PAGE_SIZE': 25,       
}
[/code]
[/li]
[li]You can now close [b]settings.py[/b] and open up [b]urls.py[/b].

Add the following url:
[code]...
url(r'^board/api/', include('django_boards.rest.urls')),
...[/code]
[/li]
[/ol]
And that's it.

Run your project and navigate to [b]/board/api/[/b], you should now see the django-rest-framework browsable API.

You may need change configurations to bend to you use case, but this should get you started.  You can visit visit documentation for django-rest-framework, which can be found on [url=http://www.django-rest-framework.org/#api-guide]their website[/url].
